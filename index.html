Part 1: Autoencoder & VAE (MNIST)

Task 1 — Basic Autoencoder

A fully connected autoencoder was implemented and trained on MNIST using three latent dimensionalities: 2, 8, and 32. The encoder compresses the 784-dim flattened input to a latent vector; the decoder reconstructs the original from it.

  • Encoder: 784 → 256 → ReLU → latent_dim
  • Decoder: latent_dim → 256 → ReLU → 784 → Sigmoid
  • Loss: Binary Cross-Entropy (BCE)
  • Optimizer: Adam, lr=1e-3, 20 epochs

Random Sampling from AE Latent Space

To test whether the latent space supports generation, random vectors were sampled from N(0,I) and decoded. This reveals a fundamental limitation of standard autoencoders.

Fig 5. Random samples, latent_dim=8. Noisy and unrecognizable.
Fig 6. Random samples, latent_dim=32. Even worse — larger space means random points are less likely to land anywhere meaningful.
Finding: Both grids produce noisy, unrecognizable outputs. The AE imposes no structure on its latent space — it only learns to compress/decompress training data. Random points almost certainly fall in regions the decoder has never seen. Notably, latent_dim=32 looks worse than dim=8: with more dimensions, training data is spread across a larger, sparser space. This directly motivates the VAE.

Task 2 — Variational Autoencoder (VAE)

A VAE adds a probabilistic bottleneck — the encoder outputs a distribution (μ, σ²) instead of a single point. The reparameterization trick allows gradients to flow through the sampling step. KL divergence regularizes the latent space toward N(0,I), enabling generation.

  • Encoder: 784 → 256 → μ (dim 2), log σ² (dim 2)
  • Reparameterization: z = μ + ε·σ, where ε ~ N(0,I)
  • Decoder: z → 256 → 784 → Sigmoid
  • Loss: ELBO = BCE(x_recon, x) + KL( N(μ,σ²) ‖ N(0,I) )

A 2D grid was swept over the latent space and each point decoded to visualize how the VAE organizes digit classes spatially. Two ranges were tested: ±1.0 (high-density region) and ±5.0 (tail regions).

Fig 10. Latent grid, range ±5.0 — center stays sharp; extreme edges become blurry/distorted.
Finding: At ±1.0, the grid shows smooth continuous transitions between digit classes (8→5→3→2→6 etc.) — the space is well-organized and continuous. At ±5.0, digits near the center remain clear but extreme values produce distorted shapes since training data never mapped to those regions. This spatial organization of digit classes emerges purely from the ELBO objective with no class labels.
Part 2: Deep Convolutional GAN (DCGAN)

Task 1 — Architecture & Implementation

A DCGAN was implemented and trained on 1,638 Pokémon images to generate 64×64 RGB images. The generator maps a 100-dim noise vector to an image via transposed convolutions; the discriminator uses strided convolutions to classify real vs. fake.

Generator

  • Input: noise vector z ∈ ℝ¹⁰⁰
  • 5 transposed conv blocks: ConvTranspose2d → BatchNorm → ReLU
  • Output: 64×64 RGB image with Tanh activation (range [−1, 1])
  • Channel progression: 100 → 512 → 256 → 128 → 64 → 3

Discriminator

  • Input: 64×64 RGB image
  • 4 conv blocks: Conv2d → BatchNorm → LeakyReLU(0.2)
  • Output: real/fake scalar via Sigmoid
  • Channel progression: 3 → 64 → 128 → 256 → 512 → 1

Discussion

Why VAEs over Basic Autoencoders for Generation?

A basic autoencoder learns a compressed representation with no constraints on latent space structure. Randomly sampling points produces garbage because the decoder has only been trained on specific encoded inputs — there is no guarantee the space between them is meaningful. VAEs fix this with KL regularization, which pushes the encoder to map all inputs toward a smooth N(0,I) Gaussian. Any random point from that distribution decodes to something plausible. The tradeoff is slightly blurrier reconstructions due to the enforced smoothness.

Mode Collapse in GANs

Mode collapse occurs when the discriminator overpowers the generator. In the LeakyReLU experiment, D loss hit ~0 by epoch 3, meaning the discriminator classified every real image as real and every fake as fake with near-certainty. The generator received vanishing gradients and all outputs converged to the same gray blob. Warning signs: D loss → 0, G loss → large constant, generated images identical regardless of input noise.

Dataset Size and GAN Stability

The counterintuitive ReLU result highlights how GAN hyperparameters are sensitive to dataset size. With only 1,638 images, the discriminator memorized real data almost instantly. LeakyReLU's non-zero negative gradient made it maximally expressive, crushing the generator. The "dying ReLU" effect accidentally weakened the discriminator just enough to maintain balance — acting as an unintentional regularizer on a small dataset. Techniques for small-dataset GANs such as differentiable augmentation or spectral normalization would likely improve results significantly.

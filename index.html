Project 4: Generative Models — VAE & DCGAN

PyTorch implementations of Variational Autoencoders and Deep Convolutional GANs

Objective

  • Implement a basic Autoencoder and Variational Autoencoder (VAE) trained on MNIST to understand latent space structure and generation.
  • Implement a Deep Convolutional GAN (DCGAN) trained on the Pokémon dataset to generate 64×64 RGB images.
  • Analyze training dynamics, compare architectural choices (LeakyReLU vs ReLU), and visualize outputs for each model.

Dataset & Setup

  • Part 1: MNIST — 60,000 grayscale 28×28 handwritten digit images.
  • Part 2: Pokémon dataset — 1,638 RGB images, resized to 64×64.
  • All models trained on NVIDIA GPU using CUDA. Framework: PyTorch with torchvision.

Part 1: Autoencoder & VAE (MNIST)

Task 1 — Basic Autoencoder

A fully connected autoencoder trained on MNIST with three latent dimensionalities: 2, 8, and 32.

  • Encoder: 784 → 256 → ReLU → latent_dim
  • Decoder: latent_dim → 256 → ReLU → 784 → Sigmoid
  • Loss: Binary Cross-Entropy (BCE)  |  Optimizer: Adam, lr=1e-3, 20 epochs

Random vectors sampled from N(0,I) and decoded — tests whether the latent space supports generation.

Both grids produce noisy, unrecognizable outputs. The AE imposes no structure on its latent space — random points fall in regions the decoder has never seen. Notably, latent_dim=32 looks worse than dim=8: with more dimensions, training data spreads across a larger, sparser space. This directly motivates the VAE.

Task 2 — Variational Autoencoder (VAE)

A VAE adds a probabilistic bottleneck — the encoder outputs a distribution (μ, σ²) instead of a single point. KL divergence regularizes the latent space toward N(0,I), enabling generation.

  • Encoder: 784 → 256 → μ (dim 2), log σ² (dim 2)
  • Reparameterization: z = μ + ε·σ, where ε ~ N(0,I)
  • Loss: ELBO = BCE(x_recon, x) + KL( N(μ,σ²) ‖ N(0,I) )

Unlike the AE, every VAE random sample produces a recognizable digit. KL regularization forces the posterior toward N(0,I), so any random point decodes to something plausible. The samples show diversity across digit classes with realistic stroke styles.

Latent Space Grid Visualization

A 2D grid swept over the latent space and each point decoded — shows how the VAE organizes digit classes spatially.

Fig 9. Range ±1.0 — smooth continuous transitions between recognizable digit classes.
Fig 10. Range ±5.0 — center stays sharp, extreme edges become blurry/distorted.
At ±1.0, smooth transitions between digit classes (8→5→3→2→6 etc.) confirm the space is well-organized and continuous. At ±5.0, extreme values produce distorted shapes since training data never mapped there. This spatial organization emerges purely from the ELBO objective with no class labels.

Part 2: Deep Convolutional GAN (DCGAN)

Task 1 — Architecture & Implementation

  • Generator: 5 transposed conv blocks (ConvTranspose2d → BatchNorm → ReLU), Tanh output. Channels: 100→512→256→128→64→3.
  • Discriminator: 4 conv blocks (Conv2d → BatchNorm → LeakyReLU(0.2)), Sigmoid output. Channels: 3→64→128→256→512→1.
  • Adam optimizer, lr=2e-4, β₁=0.5  |  50 epochs  |  Batch size 64  |  Weights initialized from N(0, 0.02)

Why did ReLU outperform LeakyReLU here? With only 1,638 images, the discriminator memorized real data almost instantly. LeakyReLU's non-zero negative gradient made it maximally expressive, crushing the generator. The "dying ReLU" effect accidentally weakened the discriminator just enough to keep training balanced — acting as an unintentional regularizer on a small dataset.

Discussion

Key Takeaways

  • Latent dimension matters. AE reconstruction improves from dim=2 (loss 0.181) to dim=32 (0.077), but larger dims make random sampling worse — more dimensions means a sparser, less structured latent space.
  • KL regularization enables generation. The VAE latent grid shows smooth digit transitions at ±1.0 — spatial organization of digit classes emerging from the ELBO objective alone, with no class labels.
  • GAN stability is dataset-size sensitive. LeakyReLU caused immediate mode collapse on 1,638 images while ReLU produced stable training — standard GAN recommendations assume much larger datasets.
  • Mode collapse warning signs. D loss → 0, G loss → large constant, outputs identical across all noise inputs, losses stop changing across epochs.
